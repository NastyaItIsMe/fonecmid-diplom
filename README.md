# Инструкция по начальной настройке программы
## Подсистема "Обслуживание клиентов"

### 1. Настройка констант для интеграции с Telegram

Для работы оповещений через Telegram-бот необходимо заполнить константы:

1. Зайдите в раздел **Обслуживание клиентов** 
2. Откройте форму **Токен управления телеграм-ботом**
3. Заполните следующие поля:
 **Токен управления телеграм-ботом:** Токен, полученный от @BotFather при создании бота - `8488236149:AAHNwPl5VFKbB4BeqE5YbDjaMVfb-o6Ftpo` 
4. Откройте форму **Идентификатор группы телеграм**
5. Заполните следующие поля:
| **Идентификатор группы телеграм:** ID Telegram-группы или канала для отправки уведомлений - `-5258945186` 

**Как получить токен бота:**
1. Найдите в Telegram бота @BotFather
2. Отправьте команду `/newbot`
3. Следуйте инструкциям для создания нового бота
4. Скопируйте полученный токен

**Как получить ID группы:**
1. Добавьте бота в группу как администратора
2. Отправьте в группу любое сообщение
3. Перейдите по ссылке: `https://api.telegram.org/bot<ТОКЕН_БОТА>/getUpdates`
4. Найдите в ответе поле `chat.id`

### 2. Настройка номенклатуры для автозаполнения

Для корректной работы автозаполнения документов реализации необходимо настроить константы с номенклатурой:

1. Перейдите в раздел **Обслуживание клиентов** 
2. Откройте форму **Номенклатура абонентская плата**
3. Заполните константы:
**Номенклатура абонентская плата:**  Номенклатура для ежемесячной абонентской платы - Выберите элемент справочника "Номенклатура", соответствующий услуге абонентского обслуживания 
4. Откройте форму **Номенклатура работы специалиста**
5. Заполните константы:
**Номенклатура работы специалиста:**  Номенклатура для работ специалистов - Выберите элемент справочника "Номенклатура", соответствующий услугам специалистов 

**Важно!** Если хотя бы одна из констант не заполнена, автозаполнение документов реализации будет недоступно и будет выдаваться ошибка.

### 3. Настройка видов договоров

Для использования функционала абонентского обслуживания:

1. Перейдите в справочник **Договоры контрагентов**
2. При создании или редактировании договора выберите вид договора **Абонентское обслуживание**
3. Для договоров этого вида станут доступны дополнительные поля:
   - **Дата начала действия**
   - **Дата окончания действия**
   - **Сумма абонентской платы**
   - **Стоимость часа работ**
4. Заполните обязательные реквизиты:
   - Для действующих договоров укажите актуальные даты
   - Для новых договоров заполните все поля

**Важно!** При создании документов **Обслуживание клиента** система будет проверять:
- Договор должен иметь вид **Абонентское обслуживание**
- Дата документа должна попадать в период действия договора
- Стоимость часа будет браться из договора

### 4. Настройка регламентного задания

Для автоматической отправки уведомлений в Telegram:

1. Перейдите в раздел **Администрирование → Обслуживание → Регламентные и фоновые задания**
2. Найдите задание **Отправка серверных оповещений клиентам**
3. Настройте расписание выполнения:
   - Рекомендуемая периодичность: **каждые 1-5 минут**
   - Для настройки нажмите **Расписание** и укажите интервал
4. Убедитесь, что задание активно (флажок **Включено**)

### 5. Настройка прав доступа пользователей

#### 5.1. Созданные роли подсистемы

В конфигурацию добавлены следующие роли:

| Имя роли | Назначение |
|----------|------------|
| **ВКМ_ЧтениеОбслуживаниеКлиента** | Права на чтение документов обслуживания |
| **ВКМ_ДобавлениеИзменениеСпециалистов** | Права на создание и редактирование справочника специалистов |
| **ВКМ_ЧтениеСпециалистов** | Права на чтение справочника специалистов |
| **ВКМ_РаботаСМассовымСозданиемАктов** | Права на массовое создание актов и работу с реализациями |
| **ВКМ_ДобавлениеИзменениеОбслуживаниеКлиента** | Права на создание и редактирование документов обслуживания |
| **ВКМ_КадровикРасчетчик** | Базовые права на редактирование документов начислений и выплат, а также просмотр отчетов по взаиморасчетам с сотрудниками |


#### 5.2. Профили групп доступа

Созданы следующие поставляемые профили групп доступа:

**Профиль "Специалист":**
**Профиль "Менеджер":**
**Профиль "БухгалтерИТФирмы":**
**Профиль "КадровикРасчетчик":**

#### 5.3. Назначение профилей пользователям

**Порядок назначения профилей:**
1. Перейдите в раздел **Администрирование → Настройка пользователей и прав → Профили групп доступа**
2. Проверьте наличие профилей: **Специалист**, **Менеджер**, **БухгалтерИТФирмы**,**Профиль "КадровикРасчетчик":**
3. Перейдите в раздел **Администрирование → Настройка пользователей и прав → Пользователи**
4. Выберите пользователя для настройки
5. В карточке пользователя укажите соответствующий профиль группы доступа
6. Сохраните изменения

### 6. Первоначальное заполнение справочников

Для начала работы заполните:

1. **Номенклатура**:
   - Создайте элемент для абонентской платы (например, "Абонентское обслуживание")
   - Создайте элемент для работ специалистов (например, "Работа")
   
2. **Сотрудники**:
   - Назначьте специалистов, которые будут выполнять работы

3. **Договоры контрагентов**:
   - Актуализируйте договоры с видом **Абонентское обслуживание**
   - Проверьте заполнение периода действия и стоимости часа
   - Укажите сумму ежемесячной абонентской платы

### 7. Проверка работоспособности

После выполнения всех настроек рекомендуется провести тестирование:

#### 7.1. Проверка создания документа обслуживания
1. Создайте документ **Обслуживание клиента** с договором абонентского обслуживания
2. Проверьте, что:
   - Документ проводится успешно
   - При изменении специалиста/даты/времени появляется запись в справочнике уведомлений
   - Регламентное задание отправляет уведомление в Telegram
   - Движения по регистру накопления **ВыполненныеКлиентуРаботы** формируются корректно

#### 7.2. Проверка печатной формы
1. В документе **Реализация товаров и услуг** нажмите кнопку **Печать → Акт об оказанных услугах**
2. Проверьте, что печатная форма содержит:
   - Номер и дату документа
   - Информацию о контрагенте, организации и договоре
   - Детализацию из табличной части
   - Итоговую сумму цифрами и прописью
   - Поля для подписей

#### 7.3. Проверка массового создания актов
1. Откройте обработку **Массовое создание актов** (раздел **Обслуживание клиентов**)
2. Укажите период (месяц)
3. Выберите Организацию
4. Нажмите **Создать**
5. Убедитесь, что:
   - Обработка выполняется в фоновом задании
   - Для каждого договора создан документ реализации
   - Созданные документы заполнены автоматически

### 8. Действия при возникновении ошибок

| Проблема | Решение |
|----------|---------|
| Не отправляются уведомления в Telegram | Проверьте токен бота и ID группы в константах |
| Не проводится документ обслуживания | Проверьте вид договора и период его действия |
| Не видно новых полей в договоре | Обновите конфигурацию базы данных |
| Нет прав на создание документа | Проверьте назначенный профиль пользователю |
| Не работает автозаполнение реализации | Проверьте заполнение констант номенклатуры |
| Не создаются акты массово | Проверьте права бухгалтера на обработку |
| В акте не заполняются работы | Проверьте наличие движений в регистре **ВыполненныеКлиентуРаботы** за месяц |

### 9. Рекомендации по эксплуатации

1. **Ежемесячно**:
   - В конце месяца запускайте обработку **Массовое создание актов**
   - Проверяйте созданные документы реализации
   - Распечатывайте и отправляйте акты клиентам

2. **Регулярно**:
   - Контролируйте работу регламентного задания по отправке уведомлений
   - Проверяйте актуальность договоров (окончание срока действия)
   - Следите за заполнением констант номенклатуры

3. **При изменении состава пользователей**:
   - Своевременно назначайте профили групп доступа
   - Удаляйте права у уволенных сотрудников

4. **Для администратора**:
   - Периодически проверяйте журнал регистрации на наличие ошибок отправки уведомлений
   - Контролируйте размер справочника уведомлений (он должен быть пустым при работающем регламентном задании)

**Техническая поддержка:**

# Инструкция по начальной настройке программы
## Подсистема "Расчёт управленческой зарплаты"

### 10. Настройка условий оплаты сотрудников

Для корректного расчета заработной платы необходимо внести информацию об условиях оплаты каждого сотрудника:

1. Перейдите в раздел **Добавленные объекты → Условия оплаты сотрудников**
2. Для каждого сотрудника заполните следующие данные на соответствующий период:

| Поле | Описание | Для кого заполняется |
|------|----------|---------------------|
| **Оклад** | Фиксированная сумма оклада в месяц | Все сотрудники |
| **Процент от работ** | Процент от выполненных клиентам работ | Только специалисты |

**Важно!**
- Данные вводятся с периодичностью **месяц**
- При изменении условий оплаты (повышение оклада, изменение процента) необходимо добавить новую запись с даты изменения
- Для специалистов процент может быть нулевым, документ **Обслуживание клиента** будет проводиться
- Если процент не установлен на дату документа, документ **Обслуживание клиента** не проводится

### 11. Настройка видов расчёта

Система автоматически настроена на следующие виды расчёта:

#### 11.1. Основные начисления
| Вид расчёта | Назначение | Правила |
|-------------|------------|---------|
| **Оклад** | Ежемесячный оклад сотрудника | Начисляется только если нет отпуска |
| **Процент от работ** | Процент от выполненных клиентам работ | Начисляется специалистам на основании документов **Обслуживание клиента** |

#### 11.2. Дополнительные начисления
| Вид расчёта | Назначение | Правила |
|-------------|------------|---------|
| **Премия** | Разовая фиксированная премия | Начисляется документом **НачислениеФиксированнойПремии** |
| **Отпускные** | Оплата отпуска | Рассчитывается по среднему заработку за 12 месяцев |

#### 11.3. Удержания
| Вид расчёта | Назначение | Правила |
|-------------|------------|---------|
| **НДФЛ** | Налог на доходы физических лиц | 13% от начисленной суммы |

### 12. Составление графика отпусков

Для планирования отпусков на год:

1. Перейдите в раздел **Добавленные объекты → График отпусков**
2. Создайте новый документ, укажите **Год**
3. В табличной части добавьте сотрудников и укажите:
   - **Дата начала отпуска**
   - **Дата окончания отпуска**
4. Сохраните документ

**Функционал контроля:**
- Строки сотрудников, у которых общая длительность отпуска превышает 28 календарных дней, подсвечиваются
- На форме **Анализ графика** (кнопка на основной форме) отображается диаграмма Ганта, позволяющая визуально оценить загрузку сотрудников и избежать одновременных отпусков всех специалистов

### 13. Порядок работы с документами по зарплате

#### 13.1. Ежемесячная процедура начисления зарплаты

1. **Ввод выполненных работ** (в течение месяца):
   - Специалисты создают документы **Обслуживание клиента**
   - Документы формируют движения по регистру **ВыполненныеСотрудникомРаботы**

2. **Начисление зарплаты** (в конце месяца):
   - Создайте документ **Начисление зарплаты**
   - Укажите месяц начисления
   - Заполните документ автоматически (рассчитаются оклады и проценты)
   - Проверьте результаты расчета
   - Проведите документ

3. **Начисление премий** (при необходимости):
   - Создайте документ **Начисление фиксированной премии**
   - Заполните список сотрудников и суммы премий
   - Проведите документ

4. **Выплата зарплаты**:
   - Создайте документ **Выплата зарплаты**
   - Укажите фактически выплаченные суммы
   - Проведите документ

#### 13.2. Расчет отпускных

1. Убедитесь, что в графике отпусков внесен отпуск сотрудника
2. При расчете зарплаты за месяц, в котором начинается отпуск:
   - Система автоматически определит период отпуска
   - Рассчитает отпускные по среднему заработку
   - Оклад за дни отпуска начисляться не будет

# Инструкция по начальной настройке программы
## Подсистема "Отчеты для контроля показателей предприятия"

### 14. Общие сведения об отчетах

В конфигурацию добавлены четыре отчета для контроля ключевых показателей деятельности предприятия:

| Отчет | Назначение | Периодичность использования |
|-------|------------|----------------------------|
| **Выработка специалистов** | Контроль фактически отработанных часов и начисленного вознаграждения | Ежедневно/Еженедельно |
| **Анализ выставленных актов** | Контроль выставленных клиентам сумм за выполненные работы | Ежемесячно |
| **Расчеты с сотрудниками** | Контроль начисленной и выплаченной зарплаты | Ежемесячно |
| **Расход запланированных отпусков** | Контроль использования отпусков | Ежегодно |

**Важно!** Все отчеты построены исключительно на данных регистров, что обеспечивает высокую производительность даже при больших объемах данных.

### 15. Отчет "Выработка специалистов"

#### 15.1. Назначение отчета
Отчет позволяет контролировать фактическую выработку специалистов и начисленное им вознаграждение в виде процента от работ, выполненных клиентам.

#### 15.2. Доступ к отчету
- **Размещение в интерфейсе:** Подсистема "Добавленные объекты" → Панель отчетов
- **Кому доступен:** БухгалтерITФирмы, Кадровики-расчетчики

#### 15.3. Параметры отчета

| Параметр | Тип | Описание |
|----------|-----|----------|
| **Период** | Стандартный период | Интервал времени, за который формируется отчет |
| **Специалист** | Справочник Сотрудники | Выбор конкретного специалиста (если не выбран - отчет по всем) |

#### 15.4. Источники данных
Отчет строится по данным регистра накопления **ВыполненныеСотрудникомРаботы**.

### 16. Отчет "Анализ выставленных актов"

#### 16.1. Назначение отчета
Отчет предназначен для контроля сумм, выставленных клиентам за выполненные работы

#### 16.2. Доступ к отчету
- **Размещение в интерфейсе:** Подсистема "Добавленные объекты" → Панель отчетов
- **Кому доступен:** Бухгалтеры

#### 16.3. Источники данных
Отчет строится по данным регистров:
- **ВыполненныеКлиентуРаботы** - для определения сумм к выставлению
- Документы **Реализация товаров и услуг** (через регистры) - для определения фактически выставленных сумм

### 17. Отчет "Расчеты с сотрудниками"

#### 17.1. Назначение отчета
Отчет позволяет контролировать взаиморасчеты с сотрудниками: начисленные суммы, выплаты и текущую задолженность.

#### 17.2. Доступ к отчету
- **Размещение в интерфейсе:** Подсистема "Добавленные объекты" → Панель отчетов
- **Кому доступен:** Кадровики-расчетчики

#### 17.3. Параметры отчета

| Параметр | Тип | Описание |
|----------|-----|----------|
| **Период** | Стандартный период | Интервал для анализа движений |
| **Сотрудник** | Справочник Сотрудники | Опционально: выбор конкретного сотрудника |

#### 17.4. Источники данных
Отчет строится по данным:
- Регистр накопления **ВзаиморасчетыССотрудниками** - остатки и обороты 
- Регистры расчета **ОсновныеНачисления**, **ДополнительныеНачисления**, **Удержания**

### 18. Отчет "Расход запланированных отпусков"

#### 18.1. Назначение отчета
Отчет позволяет контролировать использование запланированных отпусков и сравнивать план с фактом.

#### 18.2. Доступ к отчету
- **Размещение в интерфейсе:** Подсистема "Добавленные объекты" → Панель отчетов
- **Кому доступен:** Кадровики-расчетчики

#### 18.3. Источники данных
Отчет строится по данным:
- Документ **ГрафикОтпусков** (через регистры) - плановые отпуска
- Документы **Начисление зарплаты** (периоды отпусков) - фактические отпуска

### 19. Тетирование
#### 19.1. Подготовка окружения для тестирования
Для запуска тестовых сценариев необходимо:
- запустить 1С в режиме Предприятия с дополнительными параметрами запуска `/TESTMANAGER`;
- открыть внешнюю обработку Vanessa Automation;
- загрузить приложенный фича-файл `создание обслуживаний клиентов.feature` и запустить тест.  

**Техническая поддержка:**  
При возникновении вопросов по работе отчетов обращайтесь к администратору системы или разработчику конфигурации.
